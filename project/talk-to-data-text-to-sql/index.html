LLM · FastAPI · Self‑Service Analytics

Talk‑to‑Data Text‑to‑SQL

Internal platform Year: 2023 Role: Backend & LLM Engineer

FastAPI backend powered by LLMs, enabling non‑technical teams to query SQL databases in natural language and receive verified SQL queries plus visual insights, democratising access to data across the organisation.

Context

Analytics talent was a bottleneck. Business teams waited weeks for SQL queries to answer basic questions. The goal was to let non‑technical users ask questions in English and get immediate, reliable answers—without sacrificing data accuracy or exposing the database to malicious queries.

Approach

I built a "semantic layer" using FastAPI, LLMs, and a verification engine. Natural‑language questions are converted to SQL by LLMs, validated against a schema, executed, and returned with automatic chart generation.

  • Designed a FastAPI backend accepting natural‑language queries and returning JSON results.
  • Integrated LLM prompts tailored to the database schema and column descriptions.
  • Implemented a SQL validator that checks query safety before execution (no deletes, no schema changes).
  • Added a result interpreter that automatically generates charts and natural‑language summaries.
  • Built a simple frontend allowing teams to ask questions and explore results interactively.

Impact

  • Query turnaround time reduced from days to seconds for self‑service questions.
  • Analytics team freed up 25% of time previously spent on routine query requests.
  • Adoption across 15+ teams, generating 500+ queries per month.
  • Zero security incidents; all queries validated and rate-limited.

Responsibilities

Full‑stack: API design, LLM integration, SQL validation engine, database security, result formatting, and frontend prototyping.

Technologies

FastAPI Python LLMs SQL PostgreSQL React

What this illustrates

How to build safe, user‑friendly data interfaces using LLMs—combining natural language understanding with strict validation to unlock data access for non-technical teams.
